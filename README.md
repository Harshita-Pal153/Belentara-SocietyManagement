## **Project Overview**

**Belentara** is a society management system designed to streamline administrative and resident operations. It provides functionalities for both administrators and residents, including:

* Profile management
* Maintenance tracking
* Events and notices
* Service management

The system emphasizes usability, automation, and efficient record keeping.

---

## **Features**

### **Admin Panel**

* **Dashboard:** Visual updates on maintenance, services, and other society details.
* **Member Records:** Manage resident and staff information.
* **Maintenance Management:** Add charges, view transaction history, delete records, search by email.
* **Service Management:** Track and manage services.
* **Messages:** Receive and respond to resident messages.
* **Notices:** Add and manage notices for residents.
* **Events:** Add and track upcoming and past events.

### **Resident / Client Side**

* **Profile Management:** Create, update, and delete profiles.
* **Maintenance Payments:** Submit payments with screenshot uploads, view transaction history, search by flat size.
* **Notices & Events:** View society notices and events in descending order.

## **SRS Summary**

The **Software Requirement Specification (SRS)** outlines the functional and non-functional requirements of Belentara:

1. **Purpose:** Efficient management of society operations and communication.
2. **User Roles:**

   * **Admin:** Full control over society modules.
   * **Resident:** Manage profiles, submit maintenance, view notices/events.
3. **Functional Requirements:**

   * Profile creation, update, deletion
   * Maintenance tracking with reporting
   * Event and notice management
   * Payment verification via screenshots
4. **Non-Functional Requirements:**

   * Responsive and intuitive UI
   * Secure database operations
   * Efficient handling of large resident data

## **Setup Instructions**

1. Install **Java JDK** and a compatible IDE.
2. Set up **MySQL Database** and configure schema.
3. Import project into IDE.
4. Configure **Hibernate connection** with database credentials.
5. Deploy on **local server (Tomcat)** and access via browser.
6. Follow the video/demo instructions for walkthrough.
